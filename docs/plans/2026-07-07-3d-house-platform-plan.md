# 3D HOUSE FS 平台计划

## 1. 产品定位

3D HOUSE FS 的核心特色必须是三维空间分析，而不是表单工具、数据看板或普通装修模板库。用户进入系统后，第一眼应该看到完整的 3D 房屋主舞台；编辑、导入、模板、属性调整都是辅助工具。

当前优先级：
- 3D 房屋是主视觉和主操作区，占据界面中心。
- 热力图、气流图、罗盘风水必须直接叠加在 3D 房间里。
- 编辑、导入、保存、模板、属性修改退到工具栏、右侧检查器和底部信息栏。
- 风水模块只展示完整信息、空间对应关系和分析依据，不直接替用户下好坏结论。

## 2. 参考图方向

热力图参考图一：不是离散色块，而是类似地图热区的连续云图。需要有蓝、青、绿、黄、橙、红的温度渐变，并在房间地面形成自然过渡。

气流图参考图二和图三：不是每个房间一根箭头，而是类似 CFD 的流场。需要同时表达平面上的气流路径、房间内部的空间流动方向、入口/出口附近的流动面，以及强弱变化。

罗盘风水：罗盘必须是围住房子的罗盘圈，房子位于罗盘中心。九宫、方位、坐向、门向和房间落宫信息跟随同一个风水图层显隐。

## 3. 三功能显隐切换

界面只保留三个主要分析功能开关：
- 热力图：控制 3D 地面温度云图、温度点和温度色标。
- 气流图：控制地面流线、空间箭头、入口/出口流动面和强度标签。
- 罗盘风水：控制罗盘圈、八方、坐向线、门向线和九宫叠层。

墙体、编辑辅助和属性面板是工作台功能，可以保留独立辅助开关，但不能和三大分析功能混在一起造成理解负担。

## 4. 热力图实现要求

热力图必须在 3D 房间中展示，不能只在侧边栏列数据。

实现方式：
- 用动态 CanvasTexture 生成连续温度云图，贴在 3D 房间地面。
- 使用蓝色表示低温，绿色/黄色表示中间温区，橙红色表示高温。
- 用插值算法根据模拟网格和温度点生成平滑过渡。
- 在云图上加入轻量等值线感，让用户能看出温度梯度。
- 温度传感点以 3D 标记和标签显示在对应位置。
- 场景侧边显示温度色标，避免用户只能凭颜色猜。

验收标准：
- 用户能在 3D 房间地面直接看到连续热区。
- 调整温度点后，地面云图同步变化。
- 热力图开关关闭后，云图、温度点和色标一起隐藏。

## 5. 气流图实现要求

气流图必须在 3D 房间里展示，参考 CFD 平面流场和 3D 空间流线。

实现方式：
- 每个房间生成多条彩色流线，而不是单根箭头。
- 地面流线表达平面流动路径，贴近参考图二。
- 空间流线和箭头表达房间内部空气流向，贴近参考图三。
- 入口和出口附近显示半透明流动面，表达空气进入/离开区域。
- 颜色和线条粗细随气流强度变化。
- 选中房间时，该房间的流线更密、更明显。

验收标准：
- 用户能看出风从哪里进入、经过哪里、往哪里流。
- 气流不是数据列表，而是房间里的可视化流场。
- 气流图开关关闭后，流线、箭头、流动面和强度标签一起隐藏。

## 6. 罗盘风水实现要求

罗盘风水是第三个主要分析功能，应作为一个整体图层。

实现方式：
- 多层圆环围住房屋，房子位于罗盘中心。
- 外圈显示八方。
- 中圈显示坐向、门向、正北线和房屋朝向线。
- 内层叠加九宫区域、飞星编号和房间落宫信息。
- 选中房间时，同步高亮对应宫位和方向信息。

边界要求：
- 只展示信息和依据，不直接下“好/不好”的结论。
- 不把罗盘做成一堆漂浮标签。
- 罗盘风水开关关闭后，罗盘圈和九宫叠层一起隐藏。

## 7. UI 工作台原则

界面应参考 3D 软件和空间分析工具：
- 顶部栏放项目名、编辑模式、三大分析图层开关和状态。
- 左侧工具栏放选择、画墙、热力图、气流图、罗盘风水的快捷按钮。
- 中央是全尺寸 3D 主舞台。
- 右侧是属性、文件、模板检查器。
- 底部是分析摘要，不抢 3D 主舞台。

设计原则：
- 不做营销式首页。
- 不把 3D 放进小预览框。
- 不用大卡片堆满界面。
- 所有可视化优先叠加在房屋模型中。
