# AI 建模 / 专业分析 / 装修报告工作台设计

## 目标

把当前 3D HOUSE FS 从单一分析工作台升级为三大模式的软件型工作台：

1. **建模**：用户可以选择 AI 快速建模、模板建模或手动编辑。
2. **专业分析**：围绕同一个 3D 模型进行热力、气流、罗盘、动线、家具等分析。
3. **装修建议与报告**：基于户型和分析结果生成风格、材料、预算、施工建议和报告摘要。

核心原则：**3D 模型主体不切换，模式只切换工具、面板和图层。**

## 产品结构

```text
顶部：项目名 / 三大模式切换 / 图层状态
左侧：当前模式工具栏
中间：统一 3D 房屋模型
右侧：当前模式配置或属性面板
底部：当前模式输出结果
```

三大模式：

```text
建模
专业分析
装修建议与报告
```

## 建模模式

建模模式提供三类入口：

- **AI 建模**：用户输入自然语言需求，系统生成户型草案。第一版支持 OpenAI 兼容服务配置接入；未配置或调用失败时回落到本地启发式草案。
- **模板建模**：用户选择已有模板后继续修改。
- **手动建模**：保留当前画墙、门窗、设备、传感点编辑能力。

AI 建模流程：

```text
输入需求
-> AI 服务或本地规则选择模板草案
-> 几何校验
-> 进入可编辑状态
```

AI 不直接绕过几何引擎。它只选择合法模板、生成项目名和简短依据，最终仍由 `createTemplateLayout()`、`syncDerivedLayoutData()` 和校验逻辑保证模型合法。

## 专业分析模式

专业分析模式复用当前热力、气流、罗盘和房间摘要能力。

第一版保留现有三个主分析：

- 热力温度
- 气流通风
- 罗盘风水

后续扩展：

- 家具排布
- 动线效率
- 采光日照
- 隐私安全
- 综合评分

## 装修建议与报告模式

装修模式负责把分析结果转为用户可执行的装修建议。

第一版输出：

- 风格建议
- 材料建议
- 预算区间
- 施工顺序
- 避坑清单
- 报告摘要

后续可扩展为 PDF 导出、房间级材料清单和家具采购清单。

## 数据模型方向

长期建议引入：

```ts
interface HouseProject {
  layout: HouseLayout;
  furniturePlan?: FurniturePlan;
  analysisResult?: AnalysisResult;
  renovationPlan?: RenovationPlan;
  report?: ReportDraft;
}
```

第一版不新增完整项目数据模型，只在 UI 层组织三大模式，继续复用 `HouseLayout`、`SimulationResult` 和现有分析数据。

## 第一版实现范围

### 会实现

- 顶部三大模式切换。
- 建模模式底部面板：AI 建模入口、模板入口、手动编辑提示。
- AI 建模第一版：通过 OpenAI 兼容配置或本地规则根据提示词选择合适模板并加载为草案。
- 专业分析模式：沿用当前热力/气流/罗盘分析面板。
- 装修建议与报告模式：新增报告面板，输出风格、材料、预算、施工和风险建议。
- 3D 主视图在三个模式之间保持不变。

### 暂不实现

- 视觉识别 API。
- AI 直接生成任意墙体/门窗几何。
- PDF 导出。
- CAD 导入。
- 完整家具模型库。
- 自动施工报价引擎。
